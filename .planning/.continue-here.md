<decisions_made>
- Keep feed generation local-only and Atom-only.
- Use prepared feed-entry sidecars as authoritative for matching Atom entry IDs.
- Preserve unrelated existing Atom entry XML without attempting a full XML
  editor or remote feed merge.
- Keep Qt Workspace on CLI adapters for now.
- Treat `record capture` as an explicit-input capture contract, not device
  discovery.
- Treat completed publication history as explicit local result recording, not
  proof that WaystoneOS performed a remote transfer.
- Treat `publish --validate` as local readiness validation, not remote probing
  or verification.
- Keep Qt Publish-pane validation display as a read-only CLI adapter surface.
- Treat `BuildCompletedHistory` as D-Bus result-record construction only.
- Treat `SaveCompletedHistory` as a local result-recording API, not proof that
  WaystoneOS executed remote publication.
- Treat Qt completed-history display as read-only list/detail, not a UI claim
  that WaystoneOS executed publication.
- Treat `--remote-state` as caller-supplied local comparison input, not a
  remote probe or proof of remote truth.
- Treat transfer readiness as an explicit gate before any remote mutation.
- Prefer a non-mutating transfer-intent contract before any executor.
- Treat `publish --transfer-intent` and `org.waystone.Publish1.TransferIntent`
  as execution-readiness reports only; they do not contact remotes, unlock
  credentials, transfer files, delete files, verify remotes, or write completed
  history.
- Treat `publish --prepare-removable-execution` as a non-mutating local
  executor preparation contract only; it does not copy files, create
  directories, delete files, write completed history, call D-Bus, or contact
  remotes.
- Treat `publish --execute-removable` and
  `org.waystone.Publish1.ExecuteRemovable` as local/removable file-copy
  execution only; they require explicit transfer confirmation, refuse upload
  overwrites, copy through destination-directory temporary files before
  renaming into place, write completed history from executor results, and do
  not execute deletes, contact remotes, unlock credentials, probe SSH host
  keys, or verify remote results.
- Prefer removable/local execution before SSH execution once executor work is
  approved.
- Treat Qt removable destination-state export as a local comparison helper: it
  writes only a project-local manifest under `history/previews/` and does not
  execute publish transfers or inspect remote hosts.
- Do not let ordinary `--yes` imply remote delete confirmation.
- Treat sibling Waystone applications as future add-ons only; do not edit
  outside this repository unless explicitly told.
</decisions_made>
